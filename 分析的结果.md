# DarkSword 日志管理后台系统分析报告

## 一、现有系统概述

### 1.1 项目定位
**DarkSword** 是一个 iOS 漏洞利用交付框架，用于红队渗透测试操作。主要功能包括：
- HTTP 服务器交付漏洞利用 payloads
- 接收被入侵设备上传的数据（exfil）
- 记录访问日志

### 1.2 当前技术栈
| 层级 | 技术 | 版本 |
|------|------|------|
| 语言 | Python | >= 3.9 |
| HTTP 服务器 | http.server (标准库) | - |
| CLI 框架 | Click | >= 8.0 |
| 终端美化 | Rich | >= 13.0 |
| HTTP 客户端 | httpx | >= 0.25 |

### 1.3 项目结构
```
darksword/
├── darksword/           # Python 核心模块
│   ├── cli.py          # CLI 命令入口
│   ├── server.py       # HTTP 服务器（含日志记录）
│   ├── payloads.py     # Payload 管理
│   └── config.py       # 配置管理
├── payloads/            # 漏洞利用脚本（JS）
├── templates/           # Landing page 模板
├── kexploit/            # 内核漏洞利用（Objective-C）
├── exfil/               # 窃取数据存储目录
└── server.log           # 访问日志文件
```

## 二、现有日志机制分析

### 2.1 当前日志格式
当前日志记录在 `server.log` 文件中，格式为纯文本：

```
[2026-07-19 10:15:57] DarkSword server listening on http://0.0.0.0:8080
[2026-07-19 10:17:27] GET / from 192.168.31.85
[2026-07-19 10:17:27] [REQUEST] "GET / HTTP/1.1" 200 -
[2026-07-19 10:17:27] POST /upload from 192.168.31.85 (1234 bytes)
[2026-07-19 10:17:27] [EXFIL] device_uuid | credential | /path -> exfil/xxx.txt
```

### 2.2 日志类型分类
| 类型 | 关键字 | 数据内容 |
|------|--------|----------|
| 服务器启动 | `server listening` | 时间、地址、端口 |
| HTTP GET 请求 | `GET` | 时间、路径、客户端IP |
| HTTP POST 请求 | `POST` | 时间、路径、客户端IP、数据大小 |
| 数据上传 | `EXFIL` | 时间、设备UUID、类别、路径、输出文件 |
| 错误日志 | `ERROR` | 错误信息 |

### 2.3 当前日志系统的不足
1. **纯文本存储**：不支持查询、统计、过滤
2. **无结构化数据**：难以提取关键信息（设备UUID、数据类别等）
3. **无可视化界面**：只能通过命令行查看
4. **无用户权限管理**：任何人都可以访问日志
5. **无数据管理功能**：无法管理 exfil 文件

## 三、需求分析

### 3.1 功能需求
根据现有系统和用户需求，后台管理系统需要实现以下功能：

#### A. 用户与权限管理
- 用户登录/登出
- 用户角色管理（管理员、操作员）
- 权限控制（基于角色的访问控制）

#### B. 日志管理
- 访问日志查看（HTTP请求记录）
- 日志过滤（按时间、IP、路径、类型）
- 日志搜索
- 日志统计（访问量、趋势图）

#### C. 设备管理
- 设备列表（按UUID分组）
- 设备详情（访问记录、上传数据）
- 设备统计

#### D. 数据管理（Exfil）
- 窃取数据文件列表
- 数据分类管理（credentials、wifi、photos等）
- 数据下载
- 数据删除

#### E. 系统监控
- 服务器状态
- 在线设备统计
- 实时日志流

## 四、技术选型建议

### 4.1 方案对比

| 维度 | MineAdmin | Art Design Pro | 自建 |
|------|-----------|----------------|------|
| 完整度 | 完整后台系统 | 前端模板 | 需从零开发 |
| 权限管理 | 内置RBAC | 需自行实现 | 需自行实现 |
| UI组件 | Element Plus | Ant Design Vue | 自选 |
| 后端语言 | PHP (ThinkPHP) | 无后端 | Python (FastAPI) |
| 部署复杂度 | 较高 | 中等 | 较低 |
| 与现有系统集成 | 复杂 | 中等 | 简单 |

### 4.2 推荐方案

**推荐方案：自建轻量级方案（FastAPI + Vue3 + Element Plus）**

理由：
1. **技术栈统一**：后端使用 Python（与现有系统一致），降低学习成本
2. **集成简单**：直接在现有 Python 项目中扩展 API
3. **轻量级**：无需引入 PHP/MySQL 等额外依赖
4. **快速开发**：FastAPI 自动生成文档，开发效率高
5. **SQLite 数据库**：无需额外安装数据库服务

**技术栈：**
| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | FastAPI | 高性能、自动文档、类型安全 |
| 数据库 | SQLite | 轻量级、无需服务、适合单机部署 |
| ORM | SQLAlchemy | Python 标准 ORM |
| 认证 | JWT | 无状态认证 |
| 前端框架 | Vue 3 | 组合式API |
| UI组件 | Element Plus | 企业级组件库 |
| 构建工具 | Vite | 快速构建 |
| 图表 | ECharts | 数据可视化 |

### 5.2 数据流向
1. **设备访问** → HTTP Server → 记录到 server.log + 写入 logs 表 + 更新 devices 表
2. **数据上传** → HTTP Server → 保存到 exfil/ 目录 + 写入 exfil_data 表
3. **管理界面** → API 网关 → 查询数据库 → 返回数据

## 八、安全考虑

1. **JWT 认证**：使用短过期时间 + 刷新机制
2. **密码加密**：使用 bcrypt 或 passlib
3. **输入验证**：FastAPI 自动进行类型验证
4. **权限控制**：基于角色的访问控制
5. **日志审计**：记录管理员操作日志
6. **文件上传限制**：限制大小和类型
7. **CORS 配置**：限制允许的域名
8. **HTTPS**：生产环境启用

---

## 九、总结

本项目需要构建一个轻量级的后台管理系统，用于管理和监控 DarkSword 漏洞利用框架的访问日志和窃取数据。推荐使用 **FastAPI + Vue3 + Element Plus + SQLite** 技术栈，既能保持与现有 Python 项目的一致性，又能快速开发出功能完善的管理界面。
